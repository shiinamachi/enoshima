# Surface checklist

## Product states

- Default and empty
- Hover and keyboard focus
- Pressed and selected
- Disabled or unavailable
- Busy/progress and duplicate-action prevention
- Success and recoverable error
- Destructive confirmation where required

## Desktop behavior

- Internal and external monitors
- 100%, 125%, 150%, and 200% scale
- Narrow laptop width and ultrawide placement
- Pointer, touchpad, and keyboard parity
- Korean and English labels without clipping
- Reduced motion and reduced transparency
- High contrast visible focus and non-color-only state cues

## Implementation handoff

- Exact surface ID and implementation paths
- Shared token names, not only raw values
- Component dimensions and minimum hit areas
- Icon semantic names
- State transitions and cancellation behavior
- Invariants and adaptive regions
- Manual screenshot scenarios and acceptance notes
